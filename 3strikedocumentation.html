Help/Documentation

 This is a dashboard to notify associates of Cerner about the SR (Service Requests) which has not received any response from client for 
 continuous 7 working days. Moreover, it will also display an immediate report with respect to the pending updates for each SR. Currently, 
 Cerner follows a manual process to achieve 3 Strike Rule which becomes cumbersome when the queue size crosses 25+. Associates might have to 
 keep an eye on every SR to follow 3 strike rule which would then qualify for SR Closure on 8th day. To resolve this problem, this dashboard 
 will have its own back end statistics to provide an overall view of the number of SRs which needs updates/cancelation.       


 The updates page will alert the user about the pending calls/mail outbound to be sent to the client.It will display a chart with respect to 
 all SRs whether the updates is being sent to the client in the right time. If its not send that particular follow up will be marked as red 
 with a cross sign else the colour wil turn into green.


 The statistics page will ensure the user whether the process of 3 strike rule is going on correctly or not. It will have a common graph showing 
 the number of SR closed per year. For the benefit of Cerner associate it will also show the statistics of monthly inbound and monthly outbound 
 of each SR keep a record of all updates that is being sent or received.
